# 1:1 이미지 배너

## 페이지 - 1:1 이미지 소재 설정 가이드 <a href="#common" id="common"></a>

<figure><img src="../../../.gitbook/assets/1_1 이미지_영역 안내.png" alt=""><figcaption></figcaption></figure>

* 소재 심사 규정에 따라  **평어체 (반말)** 를 사용할 수 없고 맞춤법에 어긋나는 표현 혹은 신조어는 포함되지 않아야 해요.
* [업종 별 심사 가이드](../../../policy/industryguide.md)에 따라 심의필 제출이 필수인 업종의 경우, 전체 심의필 번호를 **광고 이미지 하단 중앙 정렬**로 기재해 주세요.
  * 1:1 이미지 배너 유형은 심의필 문구가 자동 반영되지 않아요 광고 이미지 내에 직접 기재하여 광고를 집행해야 해요.
  * 안내 문구 입력란(토스애즈 시스템)에 기재된 심의필 문구와 이미지 내에 심의필 문구가 다른 경우 심사 반려될 수 있어요.

{% tabs fullWidth="false" %}
{% tab title="브랜드명,브랜드 이미지" %}
- [x] &#x20;**아이콘 영역**에 입력한 브랜드 정보와 무관하게 **광고 문구** 또는 **광고 이미지**에도 브랜드 정보를 반드 기재해주세요.

* **브랜드명**
  * 공백 포함 최대 20자까지 작성할 수 있어요.
  * 랜딩 페이지에서도 확인 가능한 실제 <mark style="background-color:blue;">브랜드명</mark> 또는 <mark style="background-color:blue;">제품명</mark>만 입력가능해요.
  * 혜택 또는 정보 전달성 문구를 입력하는 경우 심사 반려될 수 있어요.
* [브랜드 이미지](logo.md) 설정 가이드를 참고하여 이미지를 설정해주세요.
  * 로고 이미지가 실제 노출 지면에 반영되었을 때 잘림 또는 왜곡 없이 잘 보여지도록 가이드를 참고하여 **이미지 여백**을 충분히 넣어주세요.

<figure><img src="../../../.gitbook/assets/로고,제품 안전 여백 (8).png" alt=""><figcaption></figcaption></figure>

* 광고 소재와 연관성이 있는 이미지를 사용해야 하고 단순 텍스트 및 기타 이미지는 사용 불가해요
* 로고의 형태가 **긴 문구**로 구성되어 있는 경우 실제 지면에서 브랜드 로고의 가독성이 떨어질 수 있어 사용 권장하지 않아요,  로고가 최대한 중앙에 위치할 수 있도록 구성해주세요.

<figure><img src="../../../.gitbook/assets/로고 가로 세로 (5).png" alt=""><figcaption></figcaption></figure>

* 실제 광고 집행 시 브랜드 이미지는 가독성을 위해 앱 버전이 다크모드인 경우 하얀색 테두리가 생길 수 있어요.

<figure><img src="../../../.gitbook/assets/다크 모드 대응 (4).png" alt=""><figcaption></figcaption></figure>

{% tab title="광고 이미지" %}
* [x] 브랜드 아이콘 영역과 동일하게 광고 문구와 이미지에 브랜드명과 브랜드 로고를 반드시 포함해주세요.

- **광고 이미지**&#x20;
  * 1200\*1200 사이즈에 jpg,png 파일을 사용해주세요.
  * **물체가 가운데에 배치된 이미지 사용을 권장해요.**
  * 모먼트 배너 이미지는 아래 여백 가이드를 참고해주세요.
    * 상하 100px / 좌우 100px
- **이미지 색상 가이드**&#x20;
  * 테두리와 배경색이 적용되기 때문에 어떤 이미지를 넣어도 잘 보여요.



<figure><img src="../../../.gitbook/assets/1) 모먼트_보더 (3).png" alt=""><figcaption></figcaption></figure>

* 광고 배지
  * 오른쪽 위에 광고 배지가 붙을 수도 있으니 중요한 요소는 피해 주세요.

<figure><img src="../../../.gitbook/assets/1-2) 모먼트 AD.png" alt=""><figcaption></figcaption></figure>

이미지 내에 문구가 **30% 이상** 차지하는 경우 심사 반려될 수 있어요.

<figure><img src="../../../.gitbook/assets/3영역 안내 (1).png" alt=""><figcaption></figcaption></figure>

* 브랜드명과 로고는 상단에 배치하고 정보 전달성 문구는 주요,보조 문구 입력란을 활용해주세요.
* 꼭 들어가야 하는 저작권,안내사항,심의필 문구는 이미지 하단 중앙 정렬하여 기재해주세요.
* **심의필 기재 가이드**
  * 광고 이미지 하단 영역에 문구가 잘리지 않도록 기재해 주세요.
    * 1200\*1200 사이즈 기준 40px 텍스트 권장
    * 800\*800사이즈 기준 20px 텍스트 권장
* **아래와 같이 광고 이미지의 가독성과 질이 떨어지는 경우 광고 집행이 제한되며, 수정을 요청할 수 있어요.**

1. 본문(주요 문구, 보조 문구)에 포함된 문구가 50% 이상 중복되는 경우
   1. 광고 효율과 소재 가독성을 위해 동일한 정보가 반복되지 않도록 구성하는 것을 권장해요.
      1. 소구 포인트는 주요/보조 문구에 기재하는 것을 권장해요.
2. 불쾌감또는 혐오감을 유발하거나 소재와 연관성이 없는 이미지를 사용하는 경우&#x20;
3. 과도한 크롭,클로즈업 이미지를 사용하거나 주요 제품을 알아볼 수 없는 분할 이미지를 사용하는 경우&#x20;
4. 토스 앱의 서비스 또는 프로모션으로 오인할 수 있는 이미지를 사용하는 경우&#x20;

<figure><img src="../../../.gitbook/assets/1_1배너_금지 이미지 (1).png" alt=""><figcaption><p>실제로 집행된 이미지 예시가 아닌, 과도한 분할 컷 및 문구 중복 <strong>예시 이미지</strong> 입니다.</p></figcaption></figure>

{% tab title="광고 문구" %}
* **주요 문구**
  * 공백 포함 최대 28자 입력 가능해요  (한 행에 최대 14자 입력 가능)&#x20;
    * 2행 모두 입력해야 하고 1행만 입력하여 심사 제출하는 경우 심사 반려될 수 있어요.
* **보조 문구**
  * 공백 포함 최대 18자 입력 가능해요.
* 주요 문구와 보조 문구에는 **중복된 혜택/ 정보를 기재할 수 없어요.**
  * 가독성을 위해 서로 다른 정보 / 혜택을 기재하거나 보조할 수 있는 문장으로 구성해주세요.
    * 광고 텍스트 색상 변경 옵션을 제공하지 않아요.
* **버튼 (cta)**
  * 버튼 문구는 공백 포함 7자 입력 가능하고 영문과 특수문자는 사용을 제한해요.
  * 버튼 문구(cta)는 아래 옵션 내에서 활용할 수 있어요.
    * 보러가기, 확인하기, 알아보기, 구경하기, 구매하기, 사러가기, 보기, 설치하기, 다운로드, 받기, 앱 열기, 시작하기
{% endtab %}

{% tab title="연결 URL " %}
* [**유저가 접속할 때 이상이 없어야 해요**](../../../policy/creativeguide.md#id-5)
  * 앱으로 랜딩된다면, 앱 미설치 유저는 앱 스토어 / 구글 플레이스토어로 랜딩될 수 있어야 해요.
  * 랜딩 과정에서 접속하는 데 5초 이상 걸리는 경우, 브릿지 페이지를 만들어 주세요.
* [**랜딩/이벤트 페이지의 내용과 소재 내용이 일치해야해요**](../../../policy/creativeguide.md#id-6)
  * 랜딩 / 이벤트 페이지에서 확인할 수 없는 혜택, 정보는 광고 승인이 불가해요.
  * 가입 또는 응모와 같이 선행 조건이 따르는 혜택은 광고 문구 내에 반드시 명시해야만 사용할 수 있어요.
* [**URL 주소는 반드시 영문으로 제출해주셔야 해요**](../../../policy/creativeguide.md#id-4)
  * 주소에 한글이 있는 경우 오류가 발생할 수 있어요.
  * 링크 내에 <mark style="color:red;">{ } ^ #</mark> 문구가 있으면 반려될 수 있어요.
    * <mark style="color:red;">^</mark> 대신 \_(언더바)를 사용해주세요.
  * http:// 는 사용이 불가하여 https:// 를 사용해주세요.
* **앱스플라이어로 생성한 원링크를 제출할 경우, 파라미터가 단축되지 않은 원본 롱링크를 제출해 주세요.**
* **숏링크/단축 URL 형태는 사용할 수 없어요. (비틀리 포함)**
* **이외의 사유로 CTA 링크에 이슈가 있다고 판단되는 경우, 링크를 재요청 할 수 있어요.**
