# 비디오 배너

## 페이지 - 비디오 소재 설정 가이드

<figure><img src="../../../.gitbook/assets/2) 세로형 영상 깃북.png" alt=""><figcaption></figcaption></figure>



* 소재 심사 규정에 따라  **평어체 (반말)** 를 사용할 수 없고 맞춤법에 어긋나는 표현 혹은 신조어는 포함되지 않아야 해요.
* [업종 별 심사 가이드](../../../policy/industryguide.md#id-1)에 따라 심의필 제출이 필수 업종인 경우, 카테고리를 선택하고 안내 문구 기재란에 작성해주시면 영상 하단 영역에 자동으로 반영돼요.

<figure><img src="../../../.gitbook/assets/비디오 배너_심의필 (1).png" alt=""><figcaption></figcaption></figure>

* 중복되거나 유사한 내용의 소재를 반복적으로 등록하는 경우 광고 집행이 제한될 수 있어요.

{% tabs %}
{% tab title="브랜드 명,브랜드 로고" %}
- [x] 브랜드 로고는 **아래 두 곳에 동일하게 사용돼요.**
  * 광고 영상 하단 브랜드 로고&#x20;
  * 함께 토스 켜고 포인트 받기 아이콘
- [x] **아이콘 영역**에 입력한 브랜드 정보와 무관하게 **광고 문구** 또는 **광고 이미지**에도 브랜드 정보를 반드 기재해주세요.

* **브랜드명**
  * 공백 포함 최대 20자 입력 가능해요.
  * 랜딩 페이지에서도 확인 가능한 실제 <mark style="background-color:blue;">브랜드명</mark> 또는 <mark style="background-color:blue;">제품명</mark>만 입력가능해요.
  * 혜택 또는 정보 전달성 문구를 입력하는 경우 심사 반려될 수 있어요.
* [브랜드 이미지](logo.md) 설정 가이드를 참고하여 이미지를 설정해주세요.&#x20;
  * 로고 이미지가 실제 노출 지면에 반영되었을 때 잘림 또는 왜곡 없이 잘 보여지도록 가이드를 참고하여 **이미지 여백**을 충분히 넣어주세요.

<figure><img src="../../../.gitbook/assets/로고,제품 안전 여백 (9).png" alt=""><figcaption></figcaption></figure>

* 광고 소재와 연관성이 있는 이미지를 사용해야 하고 단순 텍스트 및 기타 이미지는 사용 불가해요.
* 로고의 형태가 **긴 문구**로 구성되어 있는 경우 실제 지면에서 브랜드 로고의 가독성이 떨어질 수 있어 사용 권장하지 않아요,  로고가 최대한 중앙에 위치할 수 있도록 구성해주세요.

<figure><img src="../../../.gitbook/assets/로고 가로 세로-1.png" alt=""><figcaption></figcaption></figure>

* 실제 광고 집행 시 브랜드 이미지는 가독성을 위해 앱 버전이 다크모드인 경우 하얀색 테두리가 생길 수 있어요.

<figure><img src="../../../.gitbook/assets/다크 모드 대응 (5).png" alt=""><figcaption></figcaption></figure>

{% tab title="광고 영상 " %}
* [x] 광고 효과를 위해, 브랜드에서 강조하고자 하는 내용이 잘 드러나는 영상 리소스를 준비해주세요.

- 영상 리소스
  * 영상 길이 : 최소 3초 - 최대 120초 (2분) **( 15초/30초 영상 사용 권장 )**
  * 영상 지원 형식: MP4, MOV,  WebM
  * 영상 파일 크기: 500MB 미만
  * 권장 프레임 속도: 24fps 또는 30fps
  * 영상 지원 화면비 : **9:16(권장)** / 1:1 / 4:5
  * 영상 해상도 : 최소  720p 이상 (1080p 권장)&#x20;
-   영상 소리

    * 소리 여부와 관계없이 모두 광고 집행 가능하고, 소리의 유뮤와 상관없이 음소거 버튼이 노출돼요.
    * 광고 영상 업로드 단계에서 시스템에서 소리가 있는 영상인지 없는 영상인지 **원본 파일 오디오 트랙 유무**로 판단해요.



<figure><img src="../../../.gitbook/assets/비디오 배너_음소거.png" alt=""><figcaption></figcaption></figure>

* **사용자의 모바일 기기 환경에 따라 영상이 잘릴 수 있어요.**
  * 빨간색 영역 안으로 꼭 들어가야하는 브랜드 로고, 저작권, 안내사항을 기재해주세요.

<figure><img src="../../../.gitbook/assets/비디오 배너_여백 가이드.png" alt=""><figcaption></figcaption></figure>

* **아래와 같이 동영상의 가독성과 질이 떨어지는 경우 광고 집행이 제한되며, 수정을 요청할 수 있어요.**
  1. 동영상 내 텍스트, 스틸컷 등의 비중이 높은 경우
     1. 제품 또는 실사, 인물이 주로 노출되는 영상이 아닌 텍스트가 50% 이상을 차지하는 경우
        1. 영상 내에 텍스트 비중이 50% 이상을 차지하는 경우 광고 효율을 보장할 수 없어요.
        2. 소구 포인트는 주요/보조 문구에 기재하는 것을 권장해요.
  2. 동일한 creative가 연속으로 반복되는 경우
  3. 화질이 지나치게 저하되어 가독성이 떨어지는 경우
  4. 소재와 연관성이 없는 경우
  5. 지나친 섬광효과나 공포 유발로 유저에게 불편함을 주는 경우
     1. 영상 시작 10초 내 3초 이상 섬광효과, 피, 흉기, 귀신, 괴물 등의 노출이 지속되는 경우
     2. 5초이상 연속해서 섬광효과, 피, 흉기, 귀신, 괴물 등의 노출이 지속되는 경우
     3. 전체 영상의 50% 이상 섬광효과나 직접적인 공포표현을 사용하는 경우
     4. 초수가 짧더라도, 놀람을 유발할 목적으로 급작스럽게 화면 등이 전환되며 공포를 유발하는 소재가 전면에 등장하는 경우
  6. 화면이 분할되어 여러가지 영상이 동시에 노출되는 경우
     1. 3초이상 연속해서 분할컷 이미지가 노출되는 경우

<figure><img src="../../../.gitbook/assets/비디오 배너_금지 영상.png" alt=""><figcaption><p>※ 실제로 집행된 영상 예시가 아닌, <strong>가독성 또는</strong> <strong>품질이 낮은 영상 예시 이미지</strong> 입니다.</p></figcaption></figure>
{% endtab %}

{% tab title="광고 문구" %}
<figure><img src="../../../.gitbook/assets/2-2) 세로형 영상 깃북.png" alt=""><figcaption><p>문구 노출 예시</p></figcaption></figure>

* **주요 문구**
  * 공백 포함 최소 5글자 이상 최대 14자 입력 가능해요.
* **보조 문구**
  * 공백 포함 최대 18자 입력 가능해요.
* 주요 문구와 보조 문구에는 **중복된 혜택/ 정보를 기재할 수 없어요.**
  * 가독성을 위해 서로 다른 정보 / 혜택을 기재하거나 보조할 수 있는 문장으로 구성해주세요.
    * 광고 텍스트 색상 변경 옵션을 제공하지 않아요.
* **버튼 (cta)**
  * 버튼 문구는 아래 옵션 내에서 활용할 수 있어요.
    * 보러가기, 확인하기, 알아보기, 구경하기, 구매하기, 사러가기, 보기, 설치하기, 다운로드, 받기, 앱 열기, 시작하기
  * 실제 노출 지면에서 영상이 가려지거나 가독성을 낮추는 범위를 최소화하기 위해 그  외 문구와 커스텀해서 기재할 수 없어요.
* 토스 기존 서비스와 문구 톤을 맞춰야 광고 효율 (클릭률)이 올라 갈 수 있어요.
{% endtab %}

{% tab title="연결 URL" %}
* [**유저가 접속할 때 이상이 없어야 해요**](https://toss-ads.gitbook.io/guide/policy/creativeguide#id-5)
  * 앱으로 랜딩된다면, 앱 미설치 유저는 앱 스토어 / 구글 플레이스토어로 랜딩될 수 있어야 해요.
  * 랜딩 과정에서 접속하는 데 5초 이상 걸리는 경우, 브릿지 페이지를 만들어 주세요.
* [**랜딩/이벤트 페이지의 내용과 소재 내용이 일치해야해요**](https://toss-ads.gitbook.io/guide/policy/creativeguide#id-6)
  * 랜딩 / 이벤트 페이지에서 확인할 수 없는 혜택, 정보는 광고 승인이 불가해요.
  * 가입 또는 응모와 같이 선행 조건이 따르는 혜택은 광고 문구 내에 반드시 명시해야만 사용할 수 있어요.
* [**URL 주소는 반드시 영문으로 제출해주셔야 해요**](https://toss-ads.gitbook.io/guide/policy/creativeguide#id-4)
  * 주소에 한글이 있는 경우 오류가 발생할 수 있어요.
  * 링크 내에 { } ^ # 문구가 있으면 반려될 수 있어요.
    * ^ 대신 \_(언더바)를 사용해주세요.
  * http:// 는 사용이 불가하여 https:// 를 사용해주세요.
* **앱스플라이어로 생성한 원링크를 제출할 경우, 파라미터가 단축되지 않은 원본 롱링크를 제출해 주세요.**
* **숏링크/단축 URL 형태는 사용할 수 없어요. (비틀리 포함)**
* **이외의 사유로 CTA 링크에 이슈가 있다고 판단되는 경우, 링크를 재요청 할 수 있어요.**
