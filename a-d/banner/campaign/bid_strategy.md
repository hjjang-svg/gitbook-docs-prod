## 자동 입찰 클릭 최대화

<figure><img src="../../../.gitbook/assets/입찰 방식_자동 입찰 클릭 최대화 (3).png" alt=""><figcaption></figcaption></figure>

* 클릭을 최대화하기 위해 주어진 일 예산을 최대한 소진하면서도 **가장 효율적인 CPC(클릭당 단가)**&#xB97C; 얻기 위해 토스가 자동으로 입찰을 결정해요.



* 자동 입찰 클릭 최대화 기능은 **캠페인 목적 '방문 유도하기'** 에만 적용 가능하고 금액을 입력하거나 설정하지 않아요.



*   자동 입찰 클릭 최대화를 선택하여 운영하는 경우 **소진 방식도 시스템이 자동으로 운영해요.**

    * \[일반 소진] , \[빠른 소진] 옵션은 선택 및 적용이 불가해요.


* 캠페인을 만든 뒤에는 입찰 방식을 바꿀 수 있어요.
  * 광고세트 생성 후에는 \[클릭 최대화 / 직접 입찰 CPC] <> \[CPM] 사이에 입찰방식 변경이 불가해요.
  * \[자동 입찰 클릭 최대화] <> \[직접 입찰 CPC] 사이의 입찰방식 변경은 언제든 변경 가능해요.

{% hint style="info" %}
타겟팅, 예산 등 동일 조건에서 소진이 잘 된 광고세트로 실험 하였고, 직접 입찰 대비 **CPC 효율 2배 개선** 및 **높은 소진율**을 보여주는 것을 검증한 입찰 방식이예요.
{% endhint %}



* 운영  유의사항&#x20;
  * 광고세트를 켜는 시간이 설정된 시간 타겟팅보다 늦으면,  캠페인 운영 첫날에는 예산이 모두 소진되지 않을 수 있어요. 첫날부터 예산을  모두 소진하고 싶다면 **시간 타겟팅 이전에 미리 광고세트를 ON 해주세요.**
  * 일 예산, 타겟 세그먼트, 시간 타겟팅을 크게 변경하면 순간적으로 CPC 효율이 크게 변동할 수 있어요 집행 중에는 조금씩 변경하면서 효율을 확인하여 운영하는 것을 권장해요.<br>
{% endstep %}

## 자동 입찰 전환 최대화&#x20;

<figure><img src="../../../.gitbook/assets/입찰 방식_자동 입찰 전환 최대화 (3).png" alt=""><figcaption></figcaption></figure>

* 전환을 최대화하기 위해 주어진 일 예산을 최대한 소진하면서도 가장 효율적인 CPA(전환당 단가)를 얻기 위해 토스가 자동으로 입찰을 결정해요.



* 자동 입찰 전환 최대화 기능은 캠페인 목적 **‘잠재 고객 모으기’**&#xC5D0;만 적용 가능하고, 금액을 입력하거나 설정하지 않아요.



*   자동 입찰 전환 최대화를 선택하여 운영하는 경우 **소진 방식도 시스템이 자동으로 운영해요.**

    * \[일반 소진], \[빠른 소진] 옵션은 선택 및 적용이 불가능해요.


* 캠페인을 만든 뒤에는 입찰 방식을 바꿀 수 있어요.
  * 광고 세트 생성 후에는 \[전환 최대화 / 직접 입찰 CPC] <> \[CPM] 사이에 입찰방식 변경이 불가해요.
  * \[자동 입찰 전환 최대화] <> \[직접 입찰 CPC\[ 사이의 입찰방식 변경은 언제든 변경 가능해요.<br>

## 자동 입찰 tCPA

<figure><img src="../../../.gitbook/assets/입찰 방식_자동 입찰 tCPA.png" alt=""><figcaption></figcaption></figure>

* 목표 전환당 비용(tCPA)을 달성하기 위해 전환 가능성이 높은 유저에게 토스가 자동으로 입찰을 결정해요.



* 자동 입찰 tCPA는 광고주의 비용목표를 달성하는 것이 최우선이므로 **예산 소진을 반드시 보장하지 않아요.**



* 자동 입찰 tCPA 기능은 **캠페인 목적 '잠재고객 모으기'** 에만 적용 가능하고 최소 단가 4,000원 이상 최대 단가 50만원 이하로 100원 단위로 입력 가능해요.



* 자동 입찰 tCPA 으로 운영하는 경우 **소진 방식도 시스템이 자동으로 운영해요.**



*   캠페인을 만든 뒤에는 입찰 방식을 바꿀 수 있어요.

    * 광고세트 생성 후에는 \[tCPA / 직접 입찰 CPC] <> \[CPM] 사이에 입찰방식 변경이 불가해요.
    * \[자동 입찰 tCPA] <> \[직접 입찰 CPC] 사이의  입찰방식 변경은 언제든 변경 가능해요.


* 운영 유의사항&#x20;
  * 머신러닝 최적화에 시간이 소요되므로, 전환수 30건이 모일때까지는 입찰 방식을 유지하는 것을 권장해요.
  * 전환수가 모이지 않거나 예산 소진이 미비한 경우, tCPA 값을 적절하게 높여서 충분한 소진과 전환수를 확보할 수 있도록 해주세요.<br>
{% endstep %}

## 직접 입찰 CPC

<figure><img src="../../../.gitbook/assets/입찰 방식_직접 입찰 CPC (1).png" alt=""><figcaption></figcaption></figure>

* 직접 단가를 설정하여 수동으로 입찰할 수 있는 CPC 단위로 직접 입찰할 수 있어요.



* 최소 단가 200원 이상 최대 단가 10만원 이하로 설정할 수 있고 10원 단위로 입력 가능해요.



* 직접 단가를 설정하여 수동으로 입찰하고 싶다면 직접 입찰 CPC 방식으로 설정해주세요.
  * 정확한 단가 제어가 필요한 경우에 적합한 입찰 방식이에요.


{% endstep %}

## 직접 입찰 CPM

<figure><img src="../../../.gitbook/assets/입찰 방식_직접 입찰 CPM (1).png" alt=""><figcaption></figcaption></figure>

* 광고가 1,000번 노출될 때마다 CPM으로 과금되는 입찰 방식이에요.



* 직접 입찰 CPM 기능은 광고 유형 **'보드 배너'** 에만 적용 가능한 입찰 방식이에요.



* 최소 단가 1,500원 이상 최대 단가 10만원 이하로 100원 단위로 입력 가능해요.
